# sales-csv-automation

国産雑貨・日用品EC/卸売事業者向けの売上CSV取込・加工・出力を自動化するGoogle Apps Script（GAS）プロジェクト。

---

## やってきたこと

### 1. 環境構築

- `clasp` を使ってローカルのGASコードをGoogleにpushできる環境を整備
- `config.gs` に各種DriveフォルダID・スプレッドシートIDを一元管理
- `config.gs` はGit管理対象外（秘密情報保護）、`config.example.gs` をテンプレートとしてGit管理
- `.claspignore` で `config.example.gs` をpush対象から除外

### 2. マスタデータのセットアップ（setup.gs）

- DriveのCSVフォルダから5種類のマスタデータを読み込み、マスタスプレッドシートに展開
- 既存スプレッドシートがあれば再作成せず上書き（冪等性の確保）

### 3. 売上CSV取込処理（processor.gs）

#### バリデーション

| チェック種別 | 内容 |
|------------|------|
| nullチェック | 必須9項目の空欄不可 |
| 形式チェック | 日付（yyyy/MM/dd） |
| 重複チェック | ファイル内・取込済みシートとの重複注文番号 |
| マスタ参照チェック | 販売チャネル・商品コード・支払方法・店舗コード |
| enumチェック | 顧客区分（個人・法人・卸先） |
| 値チェック | マイナス・小数・単価0不可 |
| 金額整合チェック | 売上金額 = 数量 × 単価 |

#### 処理フロー
```
INPUTフォルダのCSVをファイル名順に取得
→ マスタデータと照合・バリデーション
→ 正常行 → 売上データシートに追記 + 取込済み注文番号シートに記録
→ エラー行 → エラーデータシートにエラー理由・ファイル名付きで記録
→ 処理済みCSVをOUTPUTフォルダに移動
→ 取込履歴シートに結果を記録
→ Slack通知（店舗別チャンネル or 共通チャンネル）
```

### 4. 各種CSVエクスポート（exporter.gs）

`exportAll(year, month)` を実行すると4種のCSVをEXPORT_FOLDERに出力する。

| 出力ファイル | 内容 |
|------------|------|
| `freee_YYYYMM.csv` | freee仕訳インポート形式 |
| `sales_report_YYYYMM.csv` | チャネル×店舗×カテゴリ別の売上集計 |
| `tax_accountant_YYYYMM.csv` | 税理士・会計事務所提出用詳細データ |
| `bi_YYYYMM.csv` | BI/分析ツール取込用フラットデータ |

## ファイル構成

```
src/
├── config.gs          # 各種ID・Webhook URLの設定（Git管理外）
├── config.example.gs  # config.gsのテンプレート
├── setup.gs           # マスタデータ初期セットアップ
├── processor.gs       # 売上CSV取込・バリデーション・出力
└── exporter.gs        # 各種CSVエクスポート

data/
├── master/            # マスタデータCSV（setup.gsが参照）
└── raw/               # テスト用売上CSV

SPEC.md                # CSV取込GASの設計仕様書
```

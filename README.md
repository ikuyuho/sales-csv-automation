# sales-csv-automation

国産雑貨・日用品EC/卸売事業者向けの売上CSV取込・加工・出力を自動化するGoogle Apps Script（GAS）プロジェクト。

---

## やってきたこと

### 1. 環境構築

- `clasp` を使ってローカルのGASコードをGoogleにpushできる環境を整備
- `config.gs` に各種DriveフォルダID・スプレッドシートIDを一元管理
- `config.gs` はGit管理対象外（秘密情報保護）、`config.example.gs` をテンプレートとしてGit管理
- `.claspignore` で `config.example.gs` をpush対象から除外

### 2. マスタデータのセットアップ（setup.gs）

- DriveのCSVフォルダから5種類のマスタデータを読み込み、マスタスプレッドシートに展開
- 既存スプレッドシートがあれば再作成せず上書き（冪等性の確保）

| シート名 | 内容 |
|---------|------|
| 商品マスタ | 商品コード・カテゴリ・標準単価・税区分・勘定科目 |
| 販売チャネル | 正規チャネル名・入力値（表記ゆれ対応）・チャネル種別 |
| 会計ルール | カテゴリ×チャネル種別ごとの借方/貸方勘定科目・摘要テンプレート |
| 店舗マスタ | 店舗コード・店舗名・地域・都道府県・Slack Webhook URL |
| 支払方法 | 正規名・入力値（表記ゆれ対応）・支払種別 |

### 3. 売上CSV取込処理（processor.gs）

#### バリデーション
取込時に以下をチェックし、エラー行は正常行と分離して記録する。

| チェック種別 | 内容 |
|------------|------|
| nullチェック | 必須9項目の空欄不可 |
| 形式チェック | 日付（yyyy/MM/dd） |
| 重複チェック | ファイル内・取込済みシートとの重複注文番号 |
| マスタ参照チェック | 販売チャネル・商品コード・支払方法・店舗コード |
| enumチェック | 顧客区分（個人・法人・卸先） |
| 値チェック | マイナス・小数・単価0不可 |
| 金額整合チェック | 売上金額 = 数量 × 単価 |

#### 処理フロー
```
INPUTフォルダのCSVをファイル名順に取得
→ マスタデータと照合・バリデーション
→ 正常行 → 売上データシートに追記 + 取込済み注文番号シートに記録
→ エラー行 → エラーデータシートにエラー理由・ファイル名付きで記録
→ 処理済みCSVをOUTPUTフォルダに移動
→ 取込履歴シートに結果を記録
→ Slack通知
```

#### 重複排除
- 取込済み注文番号シートで管理
- 修正CSV再投入時に未取込分のみ処理される（二重取込防止）

#### Slack通知
- 単一店舗のCSV → 店舗マスタのWebhook URLに通知
- 複数店舗混在 → 共通チャンネルに通知
- 成功時：件数＋ファイル名
- エラーあり：件数＋ファイル名＋エラーシートURL＋再投入手順

### 4. 仕様書の作成（SPEC.md）

設計中に決定した各論点をSPEC.mdにまとめた。

- バリデーション内容
- 部分失敗の扱い（正常分だけ取り込む）
- 重複排除方針
- 取込履歴管理
- エラー分の再取込フロー
- Slack通知仕様

### 5. 各種CSVエクスポート（exporter.gs）

`exportAll(year, month)` を実行すると、売上データシートから以下4種のCSVを出力する。

| 出力ファイル | 内容 |
|------------|------|
| `freee_YYYYMM.csv` | freee仕訳インポート形式（借方/貸方勘定科目・税区分・摘要を会計ルールマスタから生成） |
| `sales_report_YYYYMM.csv` | チャネル×店舗×カテゴリ別の売上集計レポート |
| `tax_accountant_YYYYMM.csv` | マスタ情報をJOINした税理士・会計事務所提出用詳細データ |
| `bi_YYYYMM.csv` | 全マスタ情報を付加したBI/分析ツール取込用フラットデータ |

## スプレッドシート構成

| スプレッドシート | 用途 |
|---------------|------|
| マスタスプレッドシート | マスタデータの管理 |
| 取込スプレッドシート | 売上データ・エラーデータ・取込済み注文番号・取込履歴 |

---

## ファイル構成

```
src/
├── config.gs          # 各種ID・Webhook URLの設定（Git管理外）
├── config.example.gs  # config.gsのテンプレート
├── setup.gs           # マスタデータ初期セットアップ
├── processor.gs       # 売上CSV取込・バリデーション・出力
└── exporter.gs        # 各種CSVエクスポート

data/
├── master/            # マスタデータCSV（setup.gsが参照）
└── raw/               # テスト用売上CSV

SPEC.md                # CSV取込GASの設計仕様書
```

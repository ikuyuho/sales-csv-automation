# 売上CSV取込 仕様書

## 概要

Google DriveのINPUTフォルダに売上CSVを配置すると、GASが自動的に取り込み・バリデーションを行い、スプレッドシートに格納する。

---

## スプレッドシート構成

### マスタスプレッドシート
| シート名 | 内容 |
|---------|------|
| 商品マスタ | 商品コード・商品名・カテゴリ等 |
| 販売チャネル | チャネルID・正規名・入力値・チャネル種別 |
| 会計ルール | カテゴリ×チャネル種別ごとの仕訳ルール |
| 店舗マスタ | 店舗コード・店舗名・Slack Webhook URL等 |
| 支払方法 | 支払方法ID・正規名・入力値 |

### 取込スプレッドシート
| シート名 | 内容 |
|---------|------|
| 売上データ | バリデーション正常の取込データ |
| エラーデータ | バリデーションエラーのデータ（エラー理由列付き） |
| 取込済み注文番号 | 重複排除用の注文番号一覧 |
| 取込履歴 | 実行ごとの取込結果ログ |

---

## バリデーション仕様

| チェック種別 | 対象 | 内容 |
|------------|------|------|
| nullチェック | 注文番号・売上日・販売チャネル・商品コード・数量・単価・売上金額・支払方法・顧客区分 | 空欄不可 |
| 形式チェック | 売上日 | yyyy/MM/dd 形式のみ |
| 重複チェック | 注文番号 | 同一ファイル内・取込済み注文番号シートと照合 |
| マスタ参照チェック | 販売チャネル・支払方法・商品コード・店舗コード | マスタスプレッドシートに存在するか |
| enumチェック | 顧客区分 | 個人・法人・卸先のいずれか |
| 値チェック | 数量・送料・手数料 | 0以上の整数 |
| 値チェック | 単価 | 1以上の整数 |
| 金額チェック | 売上金額 | 数量 × 単価 と一致するか |

---

## 処理フロー

```
1. INPUTフォルダのCSVをファイル名順に取得
2. マスタデータをスプレッドシートから読み込む
3. 各CSVを1行ずつバリデーション
   ├── 正常 → 売上データシートに追記 & 取込済み注文番号シートに追記
   └── エラー → エラー理由を付けてエラーデータシートに追記
4. 処理済みCSVをOUTPUTフォルダに移動
5. 取込履歴シートに結果を1行追記
6. Slack通知
```

---

## 部分失敗の扱い

1ファイル内にエラーが混在している場合、**正常分だけ取り込む**。エラー分は取り込まない。

---

## 重複排除

- 取込済み注文番号シートに注文番号を一覧で保持する
- CSV取込時に照合し、一致する注文番号はスキップする
- これにより修正CSV再投入時に二重取込を防ぐ

---

## 取込履歴

取込スプレッドシートの「取込履歴」シートに以下を記録する。

| 実行日時 | ファイル名 | 取込件数 | エラー件数 | ステータス |
|---------|-----------|---------|-----------|----------|

---

## エラー分の再取込フロー

```
1. 担当者がエラーデータシートを確認
2. エラー原因を修正したCSVを作成
3. INPUTフォルダに再配置
4. processCSV() が実行される
5. 取込済み注文番号シートと照合し、未取込分のみ取り込まれる
```

---

## Slack通知仕様

- 店舗マスタの `Slack Webhook URL` 列で店舗ごとのチャンネルを管理する
- 1ファイルに複数店舗が混在している場合は共通チャンネル（`config.gs` の `SLACK_WEBHOOK_URL`）に送信する
- 通知内容：
  - 正常完了時：`✅ 売上CSV取込完了：〇件取込`
  - エラーあり：`⚠️ 売上CSV取込完了：〇件取込、△件エラー` + エラーシートURL + 再投入手順
